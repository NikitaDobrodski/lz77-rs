# lz77-rs

Консольный архиватор файлов на языке Rust, реализующий алгоритм сжатия LZ77 со скользящим окном.

Проект создан как портфолио для изучения модели владения Rust, бинарного ввода-вывода и низкоуровневой обработки данных.

---

## Алгоритм

LZ77 заменяет повторяющиеся последовательности байт ссылками на более раннее вхождение в буфере.

Каждый токен кодирует тройку `(offset, length, next_char)`:

| Поле        | Описание                                    |
|-------------|---------------------------------------------|
| `offset`    | Смещение назад в буфере истории             |
| `length`    | Количество байт для копирования             |
| `next_char` | Следующий литеральный байт после совпадения |

Если совпадений нет — токен является литералом: `(0, 0, byte)`.

Поиск совпадений ускорен хеш-таблицей по 3-байтовым ключам — сложность снижена с O(n²) до ~O(n). Файлы обрабатываются потоково чанками по 4 МБ, что позволяет работать с большими файлами без загрузки всего содержимого в память.

## Использование

### Сборка

```bash
cargo build --release
```

### Интерактивный режим (TUI)

```bash
cargo run
```

Запускает терминальный интерфейс с файловым браузером. Навигация:

- `↑↓` — перемещение по списку
- буквы — поиск файлов в реальном времени
- `Tab` — ручной ввод пути
- `Enter` — выбрать файл / перейти в папку
- `Esc` — сброс поиска или назад
- `q` — выход

### Сжатие файла (CLI)

```bash
cargo run -- encode input.txt output.lz77
```

### Распаковка файла (CLI)

```bash
cargo run -- decode output.lz77 restored.txt
```

## Формат файла

Сжатые файлы используют потоковый бинарный формат:

| Поле             | Размер        | Описание                                  |
|------------------|---------------|-------------------------------------------|
| Магические байты | 4 байта       | ASCII-маркер `LZ7S` (LZ77 Streaming)      |
| Токены           | 5 байт каждый | offset(u16) + length(u16) + next_char(u8) |
| Маркер конца     | 4 байта       | `0xFFFFFFFF`                              |

Поддерживается чтение старого формата `LZ77` для обратной совместимости.

## Параметры

| Параметр        | Значение  | Описание                      |
|-----------------|-----------|-------------------------------|
| Размер окна     | 8192 байт | Глубина поиска совпадений     |
| Lookahead буфер | 64 байта  | Максимальная длина совпадения |
| Размер чанка    | 4 МБ      | Объём данных на одну итерацию |

## Заметки

Алгоритм эффективен для текстовых и структурированных данных. Для медиафайлов (WAV, MP4 и др.) сжатие неэффективно — они не содержат повторяющихся байтовых последовательностей, которые ищет LZ77.

Реализация проверена на реальных файлах — MD5-хеши оригинала и восстановленного файла совпадают.
